# Glossary

**Last Updated**: 2025-10-08
**Purpose**: Definitions of terms used throughout the Agentic Learning Framework

---

## A

### Adversarial Growth Engine
One of the 15 learning paradigms. Challenge-driven learning approach where AI creates progressively difficult problems to push learner capabilities.

### Agent
An autonomous AI entity that performs specific tasks within the system. Examples: researcher agent, coder agent, tester agent.

### Agent Collaboration
The process by which multiple agents coordinate work, share context, and avoid duplicate effort (MANDATORY-23).

### Akashic Interface
One of the 15 learning paradigms. Conceptual connection to universal knowledge through AI-mediated access to collective human understanding.

### Alexandria
Enterprise-focused implementation of the Agentic Learning Framework. Designed for organizational knowledge management and institutional learning.

### API (Application Programming Interface)
Programmatic interface for interacting with system components.

---

## B

### Batch Operations
Executing multiple operations concurrently in a single message, following the "1 MESSAGE = ALL RELATED OPERATIONS" rule.

### Beta
Software development stage where features are complete but still undergoing stabilization and testing.

---

## C

### CI/CD (Continuous Integration/Continuous Deployment)
Automated processes for testing, building, and deploying code changes.

### Claude
AI assistant created by Anthropic, used as the foundation for many project capabilities.

### Claude Code
Anthropic's official CLI tool for AI-assisted development, used extensively in this project.

### Claude Flow
Local agent coordination framework that manages topology setup and agent communication patterns via MCP.

### Cognitive OS (CognitiveOS)
Cloud-native platform implementation of the Agentic Learning Framework. Designed for scalable, global learning platforms.

### Collective Consciousness
One of the 15 learning paradigms. Shared learning experiences across multiple learners, enabling group intelligence emergence.

### Concurrent Execution
Running multiple operations in parallel within a single message, maximizing efficiency (GOLDEN RULE).

### Consciousness Evolution
Progressive development through 7 levels: Passive Reception, Active Engagement, Meta-Cognitive Awareness, Paradigm Fluidity, Consciousness Hacking, Collective Integration, Reality Programming.

---

## D

### Directive
Mandatory operational instruction for agents. 25 directives defined in CLAUDE.md (MANDATORY-1 through MANDATORY-25).

### Dissolution Protocol
One of the 15 learning paradigms. Intentional unlearning and cognitive flexibility to overcome outdated mental models.

### Dream Weaver
One of the 15 learning paradigms. Subconscious pattern integration during rest periods for deeper understanding.

---

## E

### E2B Sandbox
Isolated execution environment for running code safely. Provided by Flow Nexus platform.

### Elaborative Interrogation
Learning technique with d=0.75 effect size. Deep understanding through "why" questioning.

### Entangled Learning
One of the 15 learning paradigms. Non-local knowledge connections where understanding one concept instantly illuminates related concepts.

---

## F

### Flow Nexus
Cloud-powered platform extending Claude Flow with sandboxes, neural networks, workflows, and distributed agent orchestration.

### Fractal Hologram
One of the 15 learning paradigms. Self-similar learning patterns that repeat at all scales of understanding.

---

## G

### GOLDEN RULE
"1 MESSAGE = ALL RELATED OPERATIONS" - Mandate to batch all concurrent operations in single messages for maximum efficiency.

---

## H

### Hooks
Event-triggered commands that execute in response to tool calls. Categories: pre-operation, post-operation, session management.

---

## I

### Idempotency
Property where an operation produces the same result regardless of how many times it's executed. Critical for retry safety (MANDATORY-15).

---

## K

### Knowledge Ecosystem
One of the 15 learning paradigms. Interconnected concept networks where knowledge grows organically through relationships.

---

## L

### LTS (Long Term Support)
Version designated for extended support period (typically 18-24 months).

---

## M

### MCP (Model Context Protocol)
Protocol for agent coordination, topology setup, and communication pattern establishment. Used by Claude Flow and Flow Nexus.

### Morphogenetic Field
One of the 15 learning paradigms. Pattern recognition and emergence of higher-order understanding from component parts.

### MVP (Minimum Viable Product)
Smallest functional version of a feature that delivers value.

---

## N

### Neural Network Training
Distributed training of custom ML models across Flow Nexus sandboxes.

---

## P

### Paradigm
One of 15 distinct learning approaches in the framework. Each paradigm represents a different method of knowledge acquisition and integration.

### Paradox Engine
One of the 15 learning paradigms. Embracing contradictions to develop nuanced, multi-perspective understanding.

### Practice Testing
Learning technique with d=1.50 effect size (highest). Active retrieval strengthens memory more than passive review.

### Prometheus
Personal learning companion implementation. Designed for individual learners with 2-3 week development timeline.

### Prometheus v5
AI-native monitoring paradigm with voice/text agents, natural language processing, and autonomous response capabilities.

---

## Q

### Quantum Learning Superposition
One of the 15 learning paradigms. Holding multiple contradictory concepts simultaneously before "collapsing" to integrated understanding.

---

## R

### rUv Credits
Currency used in Flow Nexus platform for computational resources, sandboxes, and API usage.

---

## S

### Self-Explanation
Learning technique with d=0.60 effect size. Metacognitive awareness developed through explaining concepts to oneself.

### Separation of Concerns
Architectural principle distinguishing between orchestration layer (MCP/Flow) and execution layer (Task tool). See MANDATORY-6.

### Somatic Resonance Field
One of the 15 learning paradigms. Embodied learning integration connecting physical experience with conceptual understanding.

### SOLID Principles
Design principles: Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion (MANDATORY-10).

### SPARC
Methodology: Specification, Pseudocode, Architecture, Refinement, Completion. Systematic approach to test-driven development.

### Spaced Repetition
Learning technique with d=0.90 effect size. Optimal timing of review sessions maximizes long-term retention.

### Swarm
Coordinated group of agents working on related tasks with shared context and communication.

### Symbiotic Mind Mesh
One of the 15 learning paradigms. AI-human collaborative learning where both entities enhance each other's capabilities.

### Synchronicity Weaver
One of the 15 learning paradigms. Generating meaningful coincidences to facilitate learning breakthroughs.

---

## T

### Task Tool
Claude Code's primary mechanism for spawning and executing agents. Distinct from MCP coordination layer.

### TDD (Test-Driven Development)
Development approach where tests are written before implementation code.

### Technical Debt
Intentional shortcuts taken for speed that require future refactoring (MANDATORY-25).

### Temporal Learning Helix
One of the 15 learning paradigms. Spaced repetition optimization through AI-calculated intervals based on performance.

### TESLA Framework
Test-Enhanced Spaced Learning Architecture. Combines practice testing and spaced repetition for 10x learning improvement.

### Topology
Network structure defining how agents communicate. Options: mesh, hierarchical, star.

---

## U

### UX (User Experience)
Overall experience of using the system, prioritizing clarity and usability (MANDATORY-19).

---

## V

### Versioning
System for tracking software evolution. This project follows Semantic Versioning 2.0.0 (MAJOR.MINOR.PATCH).

---

## W

### Workflow
Automated sequence of tasks executed in Flow Nexus, typically event-driven with message queue processing.
